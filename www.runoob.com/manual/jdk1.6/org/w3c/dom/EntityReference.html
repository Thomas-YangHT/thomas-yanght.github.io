org.w3c.dom
接口 EntityReference

所有超级接口：
Node
public interface EntityReference
extends Node

EntityReference 节点可以用来在树中表示实体引用。注意，对字符引用和对预定义实体的引用都被视为通过 HTML 或 XML 处理器扩展，以便字符用 Unicode 等效形式表示，而不是通过实体引用表示。此外，XML 处理器可以在构建 Document 时完全扩展对实体的引用，而不是提供 EntityReference 节点。如果它确实提供这样的节点，则对于表示已知实体的引用的 EntityReference 节点而言，存在 Entity，且 EntityReference 节点的子树为 Entity 节点子树的副本。但是，在实体包含未绑定名称空间前缀时，后者不可以为 true。在此情况下，因为名称空间前缀的解析取决于实体引用的位置，可能要将 EntityReference 节点的后代绑定到不同的名称空间 URI。当 EntityReference 节点表示对未知实体的引用时，该节点没有子节点且替换值（例如由 Attr.value 使用时）为空。

对于 Entity 节点而言，EntityReference 节点及其所有后代都是只读的。

注：在扩展实体引用之后执行规范化（例如在 XML 1.0 和 XML Schema 中）时，EntityReference 节点可能导致元素内容和属性值规范化问题。

另请参见 Document Object Model (DOM) Level 3 Core Specification。
